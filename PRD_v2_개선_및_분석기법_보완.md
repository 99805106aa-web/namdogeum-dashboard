# PRD v2 — 편의사항 개선 및 분석기법 보완
**남도금형 스마트 품질 모니터링 대시보드**
작성일: 2026-03-11 | 버전: 2.0

---

## 1. 개요

### 1.1 작성 배경

기존 PRD v1은 Tab 11(주간보고) 브리핑 강화에 집중하였다.
본 PRD v2는 대시보드 **전체**를 대상으로

- 실무자가 반복적으로 느끼는 **사용 불편**을 해소하고
- 현재 빠져 있는 **품질공학 분석 기법**을 추가하여

데이터를 더 빠르게 올리고, 더 깊이 분석하고, 더 쉽게 공유할 수 있는
대시보드로 발전시키는 것을 목표로 한다.

### 1.2 현황 진단 요약

| 구분 | 현재 수준 | 주요 공백 |
|------|----------|---------|
| 데이터 입력 | Excel 드래그앤드롭 | 새로고침 시 데이터 소실, 세션 유지 없음 |
| 분석 범위 | 불량률·PPM·SPC·AI진단 | Cpk/Ppk, CUSUM/EWMA, 예측, 다변량 분석 없음 |
| 필터/설정 | 날짜·품목·목표 불량률 | 설정 저장 없음, 필터 프리셋 없음 |
| 보고 | 인쇄·PDF·Excel 내보내기 | 요약 슬라이드, 모바일 공유 없음 |
| UX | 다크모드·반응형 | 키보드 단축키, 차트 드릴다운, 비교 핀 없음 |

## 2. 사용자 페르소나

### 주 사용자 A — 품질관리 담당자 (일 1~3회 사용)
- 매일 엑셀을 올려 불량 현황을 모니터링
- **불편 포인트:** "새로고침하면 다시 올려야 한다", "필터를 매번 다시 설정한다"
- **원하는 것:** 빠른 재진입, 필터 기억, 핵심 수치 바로 보기

### 주 사용자 B — 공정/생산 엔지니어 (주 2~3회 사용)
- 특정 품목·특정 불량 유형의 원인을 파고드는 심층 분석 수행
- **불편 포인트:** "차트에서 클릭해서 해당 날짜 데이터를 보고 싶다", "Cpk가 없다"
- **원하는 것:** 차트 드릴다운, 공정 능력 지수, 예측 선

### 보조 사용자 C — 대표자/경영진 (주 1회 수신)
- 품질 현황을 5분 내 파악 후 의사결정
- **원하는 것:** 모바일에서 링크 하나로 보고서 확인, 빠른 공유

#### A-01. 세션 자동 저장 및 복원 ★★★ (최우선)

**문제:** 페이지 새로고침 또는 탭 전환 시 업로드한 Excel 데이터가 사라짐.
매일 업무 시작 시 반드시 재업로드해야 하는 불편이 발생.

**요구사항:**
- 마지막으로 분석한 데이터(JSON 직렬화)를 `localStorage`에 저장
- 페이지 로드 시 저장된 데이터가 있으면 **"이전 분석 복원하기"** 배너 표시
- 복원 시 파일명·업로드 시각·데이터 건수를 함께 표시
- 저장 용량 초과(5MB) 시 경고 토스트 및 압축 옵션 제공

```
[이전 분석 복원하기]
📄 품질데이터_2월.xlsx | 1,243건 | 어제 17:32 저장
  [복원하기]   [새로 업로드]
```

**구현 방향:**
```javascript
// 저장
localStorage.setItem('nd_last_data', JSON.stringify({ rows, filename, savedAt }));
// 복원
const saved = localStorage.getItem('nd_last_data');
if (saved) showRestoreBanner(JSON.parse(saved));
```

#### B-01. 공정 능력 지수 (Cpk / Ppk) ★★★ (최우선)

**문제:** 현재 SPC 탭은 UCL/LCL 이탈 여부만 판단함.
"이 공정이 스펙을 얼마나 여유 있게 만족하는가"를 수치로 표현하는
Cpk/Ppk가 없어 공정 개선 우선순위 결정이 어려움.

**요구사항:**
- Tab 5(SPC 관리도)에 **공정 능력 섹션** 신설
- 계산 대상: 불량률 기준 p-chart의 단기(Cpk) / 장기(Ppk) 능력
- 판정 기준:

| Cpk 범위 | 등급 | 해석 |
|----------|------|------|
| ≥ 1.67 | A+ | 매우 우수 — 6시그마 수준 |
| 1.33 ≤ Cpk < 1.67 | A | 우수 — 4~5시그마 |
| 1.00 ≤ Cpk < 1.33 | B | 보통 — 관리 지속 필요 |
| 0.67 ≤ Cpk < 1.00 | C | 불량 발생 가능 — 개선 필요 |
| < 0.67 | D | 불량 다발 — 즉시 조치 |

- 품목별 Cpk 비교 막대 차트 제공
- 주간 보고서(Tab 11)에 Cpk 요약 포함

**계산 공식:**
```
USL = targetDefectRate (목표 불량률)
LSL = 0
μ  = mean(불량률 배열)
σ  = std(불량률 배열)
Cp  = (USL - LSL) / (6σ)
Cpk = min((USL - μ)/(3σ), (μ - LSL)/(3σ))
```

#### B-02. 불량률 예측 (단기 예측선) ★★★

**문제:** 현재 추이 차트는 과거 데이터만 보여줌.
"이 추세대로 가면 다음 주 불량률이 어느 수준일까?"를 확인할 수 없음.

**요구사항:**
- 일별 불량률 추이 차트에 **예측선(Forecast Line)** 추가 (점선)
- 예측 기간: 7일 (설정 가능)
- 예측 방법: 선형 회귀 + 지수 평활(α=0.3) 앙상블 평균
- 예측 구간(신뢰도 80%): 반투명 밴드로 표현
- 예측 결과 해석 텍스트 자동 생성:

```
📈 7일 예측: 불량률 0.82% → 0.74% (▼0.08%p 개선 예상)
   신뢰 구간 [0.61% ~ 0.87%] | 예측 정확도 ★★★☆☆
```

**구현 방향:**
```javascript
function forecastLinearRegression(series, horizon = 7)
  → { forecast[], lower[], upper[], r2 }

function forecastExponentialSmoothing(series, alpha = 0.3, horizon = 7)
  → { forecast[], lower[], upper[] }
```

#### B-03. CUSUM / EWMA 관리도 ★★

**문제:** 현재 p-chart는 단일 포인트 이탈만 감지함.
작은 공정 변화가 누적될 때(drift) 초기에 감지하지 못함.

**요구사항:**
- Tab 5에 **CUSUM(누적합) 관리도** 추가
  - 목표값 대비 누적 편차를 상·하방 누적합으로 계산
  - h = 4σ, k = 0.5σ (표준 CUSUM 파라미터, 설정 가능)
  - 이탈 시 🔴 표시 + "공정 이동 감지" 알림
- **EWMA(지수 가중 이동 평균) 관리도** 추가
  - λ = 0.2 (기본값, 0.05~0.5 슬라이더로 조정)
  - 단기 변화에 민감한 관리도
- 기존 p-chart / CUSUM / EWMA 탭으로 전환 가능하게 구성

**CUSUM 공식:**
```
C⁺ᵢ = max(0, C⁺ᵢ₋₁ + (xᵢ - μ₀) - k)
C⁻ᵢ = max(0, C⁻ᵢ₋₁ - (xᵢ - μ₀) - k)
이탈 조건: C⁺ᵢ > h 또는 |C⁻ᵢ| > h
```

#### B-04. 파레토 차트 자동화 + 80/20 분석 ★★★

**문제:** 현재 불량유형별 비중은 누적 스택 차트로만 보여줌.
"어느 2~3가지 불량 유형이 전체의 80%를 차지하는가"를 즉시 알기 어려움.

**요구사항:**
- Tab 3(품목별 분석) 및 Tab 2(일자별 분석)에 **파레토 차트** 추가
  - 좌측 Y축: 불량 건수 (막대)
  - 우측 Y축: 누적 비율 (선)
  - 80% 기준선 자동 표시
  - 80% 이내 항목 자동 하이라이트 (파란색), 나머지 회색
- **자동 해석 텍스트:**

```
📊 파레토 분석 결과
  상위 2개 유형(가스·수축)이 전체 불량의 78.4%를 차지합니다.
  이 2개 유형 집중 개선 시 전체 불량 약 4/5 감소 가능.
```

- 품목별 파레토: 상위 N개 품목이 전체 불량의 X% 기여

#### B-05. 다변량 이상 탐지 (Multi-variate Anomaly Detection) ★★

**문제:** 현재 이상 탐지는 불량률 단일 지표 기반.
"생산수량이 급증하면서 동시에 불량률도 급증하는" 복합 이상은 감지 불가.

**요구사항:**
- **마할라노비스 거리(Mahalanobis Distance)** 기반 이상 탐지 추가
  - 분석 변수: 생산수량, 불량수량, 불량률, 전일 대비 변화율
  - 임계값: MD > 3.0 → 이상 플래그 (설정 가능)
- Tab 9(AI 진단)에 **다변량 이상 점수** 카드 추가
- 이상 감지 시 어떤 변수가 원인인지 기여도(%) 표시:

```
🔴 복합 이상 감지 (2026-02-18)
  마할라노비스 거리: 4.23 (임계값 3.0 초과)
  주요 기여 변수:
    생산수량 급증  ████████░░  62%
    불량률 동반 상승 ████░░░░░░  31%
    전일 대비 변화율 █░░░░░░░░░   7%
```

---

#### B-06. 상관관계 히트맵 (불량 유형 × 품목) ★★

**문제:** 현재 Tab 3의 히트맵은 색상(건수)만 표시.
"어느 품목에서 어느 불량이 통계적으로 유의미하게 많이 발생하는가"를
상관관계로 표현하지 않음.

**요구사항:**
- 기존 히트맵에 **Pearson 상관계수** 표시 모드 추가 (건수/비율/상관계수 3가지 전환)
- 상관계수 기준 자동 해석:
  - r > 0.7: "강한 연관 — 해당 품목 공정에서 이 불량 유형 집중 발생"
  - 0.4 < r ≤ 0.7: "중간 연관 — 추가 관찰 필요"
  - r ≤ 0.4: "약한 연관"
- **클러스터링** 옵션: 비슷한 불량 패턴의 품목끼리 묶어 표시

---

#### B-07. 공정 능력 트렌드 (Cpk 이력 추적) ★★

**문제:** Cpk를 단일 기간에 대해서만 계산.
"공정 능력이 개선되고 있는가, 악화되고 있는가"를 주간/월간 추이로 볼 수 없음.

**요구사항:**
- **주별/월별 Cpk 추이 차트** 추가 (Tab 5)
- Cpk 1.33 기준선(합격선) 자동 표시
- 연속 N기간 이상 Cpk < 1.0 유지 시 경고 알림
- 분기별 Cpk 요약 테이블 (목표 대비 달성 여부)

---

#### B-08. 품목별 불량 수명주기 분석 ★★

**문제:** 품목의 불량률이 "처음 생산됐을 때", "양산 안정 후", "장기 생산 시"
어떻게 변화하는지 추적하는 기능이 없음.
(배스터브 곡선 / 초기 불량 → 안정기 → 마모 불량 패턴)

**요구사항:**
- Tab 3에 **품목별 불량 추이 미니 차트(스파크라인)** 추가
  - 전체 분석 기간의 불량률 추이를 테이블 셀 안에 미니 차트로 표시
  - 추세 방향(개선/악화/안정)을 아이콘으로 표시: ▼ ▲ ─
- **배스터브 패턴 자동 감지:**
  - 초기 불량기(초반 고불량 → 감소): "초기불량 패턴 감지"
  - 마모기(후반 급증): "마모성 불량 패턴 감지"

---

#### B-09. 불량 유형 전환 탐지 (Type Shift Detection) ★

**문제:** "기존에 많던 불량 유형이 줄고, 새로운 유형이 증가하는"
공정 변화 신호를 자동으로 감지하지 못함.

**요구사항:**
- 분석 기간을 전반부/후반부로 나눠 불량 유형 비중 변화 자동 감지
- 유형 비중 변화 ±10%p 이상 시 알림
- Tab 4(알림) 또는 Tab 9(AI 진단)에 표시:

```
🔄 불량 유형 전환 감지
  가스 불량: 전반부 35% → 후반부 12% (▼23%p 감소)
  수축 불량: 전반부 18% → 후반부 41% (▲23%p 증가)
  → "가스 원인 개선 후 수축 불량 부각됨 — 후속 공정 점검 권고"
```

---

#### B-10. 목표 달성 예상일 계산 (Goal Achievement Forecast) ★

**문제:** "현재 개선 추세가 지속되면 언제 목표 불량률에 도달하는가"를
계산해주는 기능이 없음.

**요구사항:**
- 선형 회귀 기울기를 이용해 목표 불량률 도달 예상일 계산
- Tab 1 상단 배너 또는 Tab 9에 표시:

```
🎯 목표 달성 예측
  현재 불량률: 0.86%  목표: 0.80%
  현재 개선 추세 유지 시 → 약 12일 후 (3/23) 목표 도달 예상
  [예측 신뢰도: 중간 ★★★☆☆]
```

## 6. 비기능 요구사항

### 6.1 성능
- 세션 저장: 직렬화 200ms 이내, 5MB 초과 시 경고
- 예측 계산: 1,000행 기준 100ms 이내 (Web Worker 고려)
- Cpk 계산: 실시간, 추가 지연 없음 (단순 통계 연산)
- 드릴다운 모달: 클릭 후 150ms 이내 표시

### 6.2 호환성
- 기존 dashboard.html 단일 파일 구조 유지
- 외부 라이브러리 추가 없음 (기존 Plotly, xlsx.js 재활용)
- Chrome·Edge·Safari·모바일 브라우저 호환

### 6.3 다크모드 · 인쇄
- 신규 컴포넌트는 모두 `var(--*)` CSS 변수 사용
- Cpk 카드, 예측선, 파레토 차트 인쇄 시 포함

### 6.4 코드 품질
- 신규 함수는 기존 네이밍 컨벤션 준수 (`camelCase`, 한국어 변수명 병행)
- 계산 함수와 렌더링 함수 분리 (`calc*` / `render*` / `chart*`)
